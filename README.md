# CourierTrack — система учёта и трекинга курьеров

Панель администратора для управления курьерами: карточки с данными,
подтянутыми из публичного профиля Instagram (где это получается),
живая карта местоположений и страница для самих курьеров, через
которую они по своему согласию транслируют геолокацию во время смены.

## Что внутри

- `server/app.js` — главный файл сервера (Express)
- `server/db/` — база данных SQLite и скрипт создания первого админа
- `server/routes/` — API: авторизация, курьеры, геолокация
- `server/services/instagramParser.js` — попытка получить публичные данные профиля
- `server/public/` — фронтенд (страницы логина, дашборда, страница курьера)

## Требования

- **Node.js версии 22 или новее** — это важно, потому что база данных использует
  встроенный модуль `node:sqlite`, который появился в Node 22 и помечен как
  экспериментальный. Если на твоём хостинге Node старее — нужно либо обновить
  Node, либо заменить `server/db/database.js` на обычную библиотеку типа
  `better-sqlite3` (требует компиляции при установке, но это стандартная вещь).

Проверить версию: `node --version`

## Установка

```bash
cd courier-app
npm install
cp .env.example .env
```

Открой `.env` и задай свой `SESSION_SECRET` (длинная случайная строка).
Сгенерировать можно так:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

## Создание первого админа

База данных создастся автоматически при первом запуске, но аккаунт админа
нужно создать вручную (это сделано специально — чтобы доступ был только у тебя):

```bash
node server/db/seedAdmin.js твой_логин твой_пароль
```

Пароль должен быть не короче 6 символов. Можешь запустить эту команду
ещё раз с другим именем, чтобы добавить второго админа, если понадобится.

## Запуск

```bash
node server/app.js
```

Сайт будет доступен на `http://localhost:3000` (или другой порт, если
задал свой в `.env`). Зайди на `/login` и используй логин/пароль, который
создал на предыдущем шаге.

## Как это работает

### Добавление курьера
В дашборде нажми «Добавить курьера», вставь ссылку на его Instagram-профиль
и нажми «Получить данные». Система попробует скачать аватар, имя и био
со страницы профиля.

### Если Instagram заблокировал запрос
Это будет происходить регулярно — Instagram активно борется с автоматическими
запросами без авторизации (капчи, лимиты, временные блокировки IP). Когда это
происходит, форма не падает с ошибкой — она показывает предупреждение и
позволяет заполнить имя, фото и описание вручную. Это штатный режим работы,
а не временная заглушка: рассчитывай, что часть курьеров придётся заводить
вручную.

### Геолокация курьеров
Каждому курьеру при создании выдаётся уникальная ссылка вида
`/courier/<токен>`. Открой карточку курьера → «Ссылка» → отправь её курьеру
(например, в мессенджере). Когда курьер открывает её на своём телефоне и
нажимает «Начать смену», браузер запрашивает у него разрешение на доступ
к геолокации. Координаты передаются на сервер только после того, как
курьер сам разрешил доступ, и только пока страница открыта и трансляция
не остановлена. Курьер видит явный индикатор «Геолокация передаётся» и
может выключить её в любой момент.

**Это сознательное архитектурное решение, а не недоработка**: получить
геолокацию человека через его Instagram-аккаунт без его участия невозможно
и не реализовано — ни сейчас, ни как «доделать потом». Реальный трекинг
курьеров работает только через согласие самого курьера на его устройстве,
так же как это устроено у всех легальных курьерских и такси-сервисов.

## Важно перед тем, как выложить сайт в реальный интернет

1. **HTTPS обязателен.** Геолокация в браузерах работает только на HTTPS
   (кроме localhost). Без сертификата страница курьера для трекинга просто
   не будет работать на реальных телефонах. Используй любой бесплатный
   вариант (Let's Encrypt, Cloudflare) при деплое на свой домен.
2. После настройки HTTPS включи `secure: true` для cookie сессии в
   `server/app.js` (там есть закомментированная строка с подсказкой).
3. Замени дефолтный `SESSION_SECRET` на свой — без этого все сессии
   станут предсказуемыми.
4. Папка `data/` (база данных) и `server/uploads/` (фото курьеров) должны
   регулярно бэкапиться — это единственное место хранения твоих данных.

## Структура базы данных

- `admins` — учётные записи администраторов (паролями хранятся как bcrypt-хеши)
- `couriers` — карточки курьеров: имя, данные из Instagram, токен для трекинга, статус
- `locations` — история координат каждого курьера (растёт со временем; при
  большом масштабе стоит периодически архивировать старые записи)
